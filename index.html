Словарь для компьютерщиков

А

  • Алгоритм: Набор инструкций для решения задачи компьютером.
  • Архиватор: Программа для сжатия файлов.
  • Апгрейд: Обновление оборудования или ПО для улучшения работы.
  • Автозагрузка: Программы, запускающиеся при старте системы.
  • Админка: Интерфейс для управления сайтом или сервером.

Б

  • Бэкенд: Серверная часть разработки веб-сайтов.
  • Бинарник: Исполняемый файл в двоичном формате.
  • Баг: Ошибка в коде, нарушающая работу программы.
  • Бэкап: Резервная копия данных.
  • Браузер: Программа для просмотра веб-страниц.
  • Бот: Программа, выполняющая автоматические действия.

В

  • Вебка: Веб-камера для видеосвязи.
  • Виджет: Небольшой интерфейсный элемент приложения.
  • Виртуалка: Виртуальная машина для эмуляции системы.
  • Валидация: Проверка корректности данных.
  • Винт: Жесткий диск компьютера.

Г

  • Глюк: Непредвиденный сбой в программе.
  • Гит: Система контроля версий для кода.
  • Гига: Сокращение от гигабайта.
  • Гаджет: Маленькое устройство или приложение.

Д

  • Дебаг: Поиск и исправление ошибок в коде.
  • Деплой: Развертывание программы на сервере.
  • Дамп: Снимок памяти программы для анализа.
  • Движок: Основной компонент программы или игры.
  • Дроп: Удаление данных или файлов.

Е

  • Единица: Сленг для обозначения одного бита или байта.

Ж

  • Железо: Аппаратное обеспечение компьютера.
  • Журнал: Лог событий программы или системы.

З

  • Залипуха: Мелкая ошибка, вызывающая неудобства.
  • Заплатка: Быстрое исправление ошибки в программе.

И

  • Интерфейс: Способ взаимодействия с программой.
  • Итерация: Повторение цикла в коде.

К

  • Кэш: Временное хранилище для ускорения доступа.
  • Кодер: Программист, пишущий код.
  • Костыль: Временное решение проблемы в коде.
  • Краш: Аварийное завершение программы.

Л

  • Лаг: Задержка в работе программы или сети.
  • Локалка: Локальная сеть.

М

  • Макрос: Автоматизированная последовательность команд.
  • Модем: Устройство для подключения к интернету.

Н

  • Нуб: Новичок в программировании или играх.

О

  • Оверхед: Дополнительные затраты ресурсов.
  • Очистка: Удаление ненужных данных или кэша.

П

  • Патч: Обновление для исправления ошибок.
  • Пинг: Проверка скорости соединения с сервером.
  • Пиксель: Минимальная единица изображения.

А

  • Абстрактный тип данных: Модель данных, определенная поведением.
  • Абстрактный метод: Метод без реализации, только сигнатура.
  • Абстракция: Упрощение путем удаления деталей.
  • Алгоритм: Инструкции для решения вычислительных задач.
  • Агрегатная функция: Функция для группировки данных.
  • Анализ сложности: Оценка ресурсов алгоритма.
  • Архитектура ПО: Структура программной системы.
  • Ассоциативный массив: Тип данных с парами ключ-значение.
  • Автомат: Математическая модель вычислений.
  • Авторизация: Проверка прав доступа.

Б

  • База данных: Организованный набор данных.
  • Байесовская сеть: Модель для вероятностных выводов.
  • Бинарное дерево: Структура с узлами до двух детей.
  • Бит: Единица информации (0 или 1).
  • Блокчейн: Распределенная база данных транзакций.
  • Булева алгебра: Алгебра с операциями истина/ложь.
  • Буфер: Временное хранилище данных.

В

  • Виртуализация: Эмуляция аппаратного обеспечения.
  • Временная сложность: Время выполнения алгоритма.
  • Вычислительная модель: Абстракция вычислительного процесса.

Г

  • Гипертекст: Текст с ссылками на другие данные.
  • Граф: Структура данных из узлов и ребер.

Д

  • Двоичный поиск: Поиск в отсортированном массиве.
  • Декомпозиция: Разбиение задачи на части.
  • Детерминированный автомат: Автомат с однозначными переходами.

И

  • Идентификатор: Уникальное имя объекта в программе.
  • Инкапсуляция: Сокрытие внутренней реализации объекта.
  • Интерпретатор: Программа, выполняющая код построчно.

К

  • Кэширование: Хранение данных для быстрого доступа.
  • Компиляция: Преобразование кода в машинный язык.
  • Криптография: Методы защиты данных.

М

  • Машинное обучение: Алгоритмы, обучающиеся на данных.
  • Многопоточность: Выполнение нескольких задач одновременно.

Н

  • Наследование: Механизм передачи свойств в ООП.
  • Нормализация: Устранение избыточности в базе данных.

О

  • Объект: Экземпляр класса в программировании.
  • Оптимизация: Улучшение производительности кода.

П

  • Параллелизм: Одновременное выполнение задач.
  • Полиморфизм: Разные реализации одного интерфейса.
  • Протокол: Правила обмена данными в сети.

Р

  • Рекурсия: Вызов функции самой себя.

С

  • Сжатие данных: Уменьшение объема данных.
  • Синтаксис: Правила написания кода.

Т

  • Тестирование: Проверка работоспособности программы.

У

  • Условный оператор: Код, зависящий от условия.
